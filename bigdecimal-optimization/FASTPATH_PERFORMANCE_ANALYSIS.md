## 快速路径优化预期

### 优化原理

快速路径主要优化：
1. **避免 longMultiplyPowerTen 的溢出检查**
2. **避免 BigInteger 分配**
3. **减少分支预测失败**

### 预期性能提升（保守估计）

| 场景 | 基线 ns/op | 预期 ns/op | 预期提升 |
|------|-----------|-----------|---------|
| Scale 相等 | 11.76 | 9.00 | **23%** |
| scaleDiff > 0 | 9.13 | 8.00 | **12%** |
| scaleDiff < 0 | 8.93 | 8.00 | **10%** |

### 为什么提升有限？

1. **JIT 已经优化**：简单除法已经被 JIT 内联和优化
2. **分支预测成功**：scale 检查分支容易被预测
3. **缓存友好**：紧凑的 long 操作缓存命中率高

## 附录：性能测试方法

```java
// 标准性能测试模式
private static final int WARMUP = 100_000;
private static final int ITERATIONS = 10_000_000;
private static volatile BigDecimal blackhole;

// Warmup
for (int i = 0; i < WARMUP; i++) {
    blackhole = operation();
}

// Measure
long start = System.nanoTime();
for (int i = 0; i < ITERATIONS; i++) {
    blackhole = operation();
}
long end = System.nanoTime();

double nsPerOp = (end - start) / (double) ITERATIONS;
```

**注意事项**：
- 使用 volatile 变量防止优化
- 足够的 warmup 迭代
- 多次运行取平均值
- 考虑 GC 影响
